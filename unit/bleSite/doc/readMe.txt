
控制：
    获取扫描结果
    绑定设备
    删除设备
    控制设备
    获取设备列表
    获取状态列表

消息
    扫描结果消息
    单个绑定成功消息
    绑定结束消息
    单个设备解绑消息
    设备状态变更消息

绑定流程：
    1. 获取扫描结果：发布扫描结果消息
    2. 绑定设备：
        1. 发布扫描结果消息
        2. 绑定设备后，更新地址列表、设备列表、状态列表
        3. 发布单个设备绑定成功
        4. 全部成功绑定成功后
                * 同步请求，更新配置站点白名单
                * 发布全部绑定成功

解绑流程；
    1. 删除地址列表、设备列表、状态列表相应条目
    2. 同步请求，删除配置站点白名单中相应条目
    3. 发布设备解绑消息

控制流程：
    1. 接收控制指令
    2. 解析返回状态消息，
    3. 更新状态列表、发布状态变更消息


 * 将白名单相关设备的属性信息，更新到设备列表条目中


白名单，蓝牙设备列表同步方式
    添加设备、删除设备、定时更新：
    	更新设备管理站点设备列表(必定会导致设备列表更新)
    	更新白名单（设备新增，或设备删除，才会导致白名单被更新）

    白名单被修改
        1. 更新本地蓝牙站点列表（只会更新设备属性信息）
        2. 更新设备站点设备管理列表


1. 通过主从站点找到设备管理站点的ip和端口号
2. 添加、删除设备，会向设备管理站点发送白名单更新请求。 此时synergy会重新获取所有设备，并和白名单比较对白名单进行更新。
3. 定时向设备管理站点发送白名单更新请求
4. 白名单被修改后，会发送白名单被修改消息。蓝牙站点收到消息后，会自动更新设备属性。并通知设备管理站点进行设备更新
